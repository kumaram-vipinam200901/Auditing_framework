# `naf-audit` — Can we *catch* a language model that breaks its copyright promise?

> A statistical auditor for the Near-Access-Freeness (NAF) guarantee. Drop in two HuggingFace models, get back a high-confidence lower bound on the worst-case KL leak — and a concrete prompt that exposes it.

## The one-paragraph pitch

Vyas, Kakade and Barak (ICML 2023) introduced **Near-Access-Freeness**: a deployed language model `P★` is *K*-NAF if, for every prompt `x` and every copyrighted work `C`, the KL divergence to a baseline that **never saw `C`** stays below `K`. The bound is the formal underpinning of recent copyright-safety claims (anchored decoding, DP-finetuned models, …). **No one has, until now, *tested* the bound.**

This repository is that test. We turn the NAF claim into a one-sided hypothesis, derive a per-prompt **Empirical Bernstein lower bound** on the KL, wrap it in an **evolutionary worst-case search**, and report a single number — the **Copyright Leakage Score (CLS)** — that says "audited: violated by ≥ CLS, with confidence ≥ 1−δ" or "audited: no violation found at this budget."

The audit is grey-box: it needs sampling from the deployed model and the safe model's log-prob of those samples. Nothing else. No retraining. No gradient access. No closed-form KL.

## Headline results (full numbers in `finalreport/main.pdf`)

| Question | Answer | Where |
|---|---|---|
| Does the auditor ever lie? | **0 / 75 000** Type-I false positives (nominal `δ=0.05`). | §3 |
| Does it catch real violators? | **88 %** at 50 %-budget margin with `n=1000` samples; **95 %** at 25 %-margin with `n=5000`. | §3 |
| Is anchored decoding actually safe? | **0 / 120** violations across 4 budgets × 30 prompts. Compliant, and *tight* at `K=0.5` (max KL = 0.500 exactly). | §4 |
| Does `ε`-DP imply `ε²/2`-NAF in practice? | **No.** A paired leave-one-out audit of an `ε`-DP fine-tune routinely exceeds `ε²/2`. The implication holds only at impractically tight `ε`. | §5 |
| Does evolutionary search beat random prompts? | Yes, on every non-trivial pair of 8 tokeniser-matched model families. | §6 |

## How it works, in 30 seconds

```
              P_risky(·|x)               P_safe(·|x)
                  │                          │
                  ▼                          ▼
        sample Y₁..Yₙ ~ P_risky      score Yᵢ under both
                  │                          │
                  └──────────┬───────────────┘
                             ▼
                    Zᵢ = log P_risky(Yᵢ|x) − log P_safe(Yᵢ|x)
                             │
                             ▼
            Empirical Bernstein lower bound on E[Z] = KL(P_risky‖P_safe)
                             │
                             ▼
              ┌──────────────────────────────┐
              │  LB > K  ⇒  K-NAF FALSIFIED  │
              │            (confidence 1−δ)  │
              └──────────────────────────────┘
                             ▲
                             │
              evolutionary search over prompts x
              maximises LB ⇒ Copyright Leakage Score
```

Two algorithms, both formally stated in the paper (§2):
- **Algorithm 1** — the Bernstein auditor: a six-line procedure that turns `n` samples into `(Z̄ₙ, Vₙ) ↦ t_n(δ) ↦ LB̂`.
- **Algorithm 2** — evolutionary adversarial prompt search: elite-retention + mutation operators (verbatim instruction, quote-wrap, format, style, crossover).

## Use it in your own project — 9 lines

```python
from src.config import HardwareConfig
from src.models import load_model_pair
from src.kl_estimator import estimate_kl

hw = HardwareConfig(device="auto", mixed_precision="fp16")
risky, safe = load_model_pair("gpt2-medium", "gpt2", hw)

r = estimate_kl(
    prompt_text="Call me Ishmael. Some years ago—",
    risky=risky, safe=safe,
    n_samples=1000, delta=0.05, claimed_K=1.0,
)
print(f"LB={r.lower_bound:.3f}   violates? {r.violates}")
```

For a full worst-case audit (CLS via the evolutionary loop): `python -m src.run_audit --config configs/fast.yaml`.
For auditing an anchored-decoded model: swap `risky` for `AnchoredDecodingModel(risky, safe, K=...)` — same API.

## What it is, and what it isn't

**It is:**
- A per-prompt, distribution-free, finite-sample lower bound on KL with formal `1−δ` coverage.
- A practical worst-case audit driver that returns a *named adversarial prompt*, not just a score.
- A drop-in for any two HuggingFace causal LMs that share a tokenizer.
- A reproducibility artifact: every figure and table in `finalreport/main.pdf` is regenerated by a script in `src/`.

**It is not:**
- A *certificate*. Failing to falsify `K`-NAF is not a proof that `K`-NAF holds — it means the auditor with this `n`, this `δ`, and these prompts could not find a counterexample.
- A near-boundary test. At ≥ 95 % of the true budget, power saturates at our `n ≤ 5000`. Use parametric estimators (and give up formal coverage) if you need that regime.
- A black-box auditor. We need log-probs of the safe model on risky-model samples. Closed-API models (GPT-4 et al.) are out of scope.
- A test for tokeniser-mismatched pairs. A byte-level common projection is left for future work.

## Reproducing the paper

```bash
pip install -r requirements.txt
python tests/test_smoke.py                                            # ~ 2 min, CPU

# §3 Calibration ─ power & Type-I (~ 30 s / seed on a single GPU)
python -m src.power_experiment   --seed 42 --out runs/power_full
python -m src.plot_power         --in  runs/power_full

# §4 Anchored decoding tightness (~ 2 min)
python -m src.anchored_tightness --seed 42 --out runs/anchored_tightness
python -m src.plot_anchored      --in  runs/anchored_tightness

# §5 DP → NAF, paired leave-one-out (~ 30 min / ε)
python -m src.dp_naf_loo         --epsilons 1 2 4 8 --out runs/dp_naf_full
python -m src.plot_dp_naf        --in  runs/dp_naf_full

# Multi-seed reproducibility table (Table 2 in the paper)
python -m src.aggregate_seeds                              # → runs/repro_table.tex
```

Re-running each experiment with `--seed 43` and `--seed 44` gives the per-seed columns of Table 2. End-to-end runtime fits within a single laptop-GPU evening session.
